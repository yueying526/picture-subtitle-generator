# 图片字幕生成器 (Image Subtitle Generator) 产品需求文档 (PRD)

## 1. 项目概述
本项目旨在开发一个基于 Web 的图片字幕生成工具，允许用户上传静态图片，并根据输入的文字内容，生成类似“电影台词长图”的拼接效果。核心功能是将图片底部区域作为背景，针对每一行字幕进行垂直堆叠拼接，形成连贯的字幕长图。

## 2. 用户界面 (UI) 需求

### 2.1 整体布局
页面应简洁明了，分为上下两部分或左右布局：
*   **控制区**：包含文件上传、参数设置、文本输入、操作按钮。
*   **预览区**：实时或点击生成后显示处理结果。

### 2.2 功能模块细节

#### 2.2.1 标题
*   显示应用名称：“图片字幕生成器”。

#### 2.2.2 文件上传
*   **组件**：文件选择器按钮。
*   **标签**：“选择文件”。
*   **反馈**：选择文件后，显示文件名（例如 `image.png`）。
*   **支持格式**：常见图片格式（JPG, PNG, WebP 等）。

#### 2.2.3 参数设置栏
一行内排列以下设置项：
1.  **字幕高度 (px)**
    *   类型：数字输入框。
    *   默认值：`40`。
    *   作用：定义每一行字幕切片的高度（即从原图底部截取的高度）。
2.  **字体大小 (px)**
    *   类型：数字输入框。
    *   默认值：`20`。
    *   作用：控制字幕文字的大小。
3.  **字体颜色**
    *   类型：颜色选择器 (Color Picker)。
    *   默认值：白色 (`#FFFFFF`)。
4.  **轮廓颜色** (描边)
    *   类型：颜色选择器 (Color Picker)。
    *   默认值：黑色 (`#000000`)。
    *   作用：为字体添加描边以增加对比度。

#### 2.2.4 字幕文本输入
*   **组件**：多行文本域 (Textarea)。
*   **占位符/默认内容**：用户可输入多行文字。
*   **逻辑**：文本域中的**每一行**对应生成图中的**一条字幕条**。空行应被忽略或生成空字幕条（视具体实现而定，建议忽略空行）。

#### 2.2.5 操作按钮
*   **生成字幕图片** (绿色按钮)：点击后触发图像处理逻辑，在预览区渲染结果。
*   **保存图片** (绿色按钮)：点击后将当前生成的图片下载到本地。

#### 2.2.6 预览区域
*   展示处理后的最终长图。
*   宽度应自适应屏幕或保持原图比例缩放展示。

## 3. 功能逻辑 (核心算法)

### 3.1 图像拼接逻辑 (关键)
为了实现“电影台词拼接”效果，生成逻辑如下：

1.  **输入**：
    *   原图 `SourceImage` (宽 `W`, 高 `H`)。
    *   字幕文本列表 `TextList` (共 `N` 行)。
    *   配置参数：字幕高度 `h` (Subtitle Height)。

2.  **处理步骤**：
    *   **第一部分 (顶部)**：绘制完整的原图 `SourceImage`。此时，第一行字幕 (Line 1) 应直接绘制在原图的底部区域（距离底部一定边距，垂直居中于底部的 `h` 区域内）。
    *   **后续部分 (拼接)**：
        *   从第 2 行字幕开始 (Line 2 到 Line N)，对每一行进行循环处理：
        *   **截取背景**：从原图 `SourceImage` 的底部截取高度为 `h` 的切片。即截取区域为 `(0, H-h)` 到 `(W, H)`。
        *   **绘制文字**：在截取的切片上绘制对应的字幕文本。
        *   **拼接**：将处理好的切片依次垂直追加到图像下方。

3.  **最终输出尺寸**：
    *   宽度：`W` (与原图一致)。
    *   高度：`H + (N - 1) * h`。
    *   *(注：如果 N=0，则只显示原图；如果 N=1，显示原图+底端文字，高度 H)*

### 3.2 文字渲染规则
*   **位置**：水平居中 (Text Align Center)，垂直居中于当前的切片高度 `h` 内。
*   **样式**：应用用户选择的字体大小、颜色。
*   **描边**：应用用户选择的轮廓颜色，通常使用 Stroke 描边以确保在复杂背景上清晰可见。

## 4. 技术栈建议
*   **前端**：HTML5, CSS3, JavaScript (原生或框架均可)。
*   **图像处理**：HTML5 Canvas API。
    *   使用 `drawImage()` 方法进行切片和拼接。
    *   使用 `fillText()` 和 `strokeText()` 进行文字绘制。
    *   使用 `toDataURL()` 或 `toBlob()` 实现图片导出下载。

## 5. 异常处理
*   **未选择图片**：点击生成时提示“请先上传图片”。
*   **无文本**：点击生成时提示或仅输出原图。
*   **图片加载失败**：处理无效图片文件的错误。

## 6. 验收标准
*   上传图片后能正常显示文件名。
*   调整参数（高度、大小、颜色）后，点击生成能看到明显变化。
*   输入多行文本时，生成的图片应在原图下方正确拼接对应数量的条带。
*   拼接的条带背景应严格一致（均为原图底部切片），且无缝连接。
*   点击保存能下载 PNG 或 JPG 格式的长图。
